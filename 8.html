Touch

Button hit areas

Ideally, buttons should be between 44px and 57px on a standard screen. This allows enough area for the average fingertip to easily activate a button.

Gestures

By allowing users to tap, pinch, or swipe the screen you will find that your users are more engaged. It will be easier for users to experience the results of their actions.

Best Practices

  • Don't make users pinch to zoom

  • Keep users in a single browser window

  • Don't just stack, resize too

  • Let users go back (the back button is harder to access)

  • All actions should show feedback (let the user know they just did something as browser response times may be slow and there is no default browser loading animation)

  • Build off what users know

Navigation

Display All



Menu items all fit in a small space at the top of the screen

Pros - very scannable, no digging

Cons - only works with small menu, takes up screen real estate, small touch area (bad for fat fingers)

Expanding Menu



The hamburger icon or a menu link is all that appears in the header. Tapping the icon shows a list of main menu items.



Pros - can fit larger menu, scrollable

Cons - too many menu options and sub menu options could push the content further down the screen making it frustrating and awkward to use.

Flyout Menu



Navigation slides out in a column from the side of the screen



Pros - can see page layered underneath, can fit larger menu, scrollable

Cons - too long menues can make it frustrating and awkward to use.
